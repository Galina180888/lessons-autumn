Основні підходи до верстки

У верстці є кілька основних підходів. Mobile-First означає спочатку створення мобільної версії сайту, а потім адаптацію під планшети та десктопи – це актуально, коли більшість трафіку йде зі смартфонів. Desktop-First працює навпаки: спочатку розробляється десктопна версія, а потім додаються стилі для менших екранів – підходить для складних веб-застосунків.

Також існує Adaptive Design, де створюються окремі версії сайту для кожного типу пристроїв, і Responsive Design, який використовує гнучкі сітки, медіа-запити та відсоткові розміри, щоб сайт динамічно підлаштовувався під будь-який екран. У сучасній розробці найчастіше комбінують Mobile-First та Responsive Design для універсального та зручного рішення.

Mobile-First (Мобайл-Фьорст)

Суть: спочатку верстають мобільну версію сайту, а потім додають стилі для планшетів і десктопів через медіа-запити.

Як застосовується:

  • Коли більшість користувачів заходять із мобільних пристроїв.
  • Для сучасних сайтів, де важлива швидкість завантаження.
  • Якщо потрібно оптимізувати сайт для SEO (Google віддає перевагу mobile-friendly сайтам).

Приклад коду (спочатку стилі для мобільних, потім для ширших екранів):

                        
                        body {
                            font-size: 16px;
                        }
                        
                        @media (min-width: 768px) {
                            body {
                                font-size: 18px;
                            }
                        }
                        
                        @media (min-width: 1024px) {
                            body {
                                font-size: 20px;
                            }
                        }

Desktop-First

Суть: спочатку верстають десктопну версію, а потім адаптують для планшетів і мобільних, додаючи стилі через max-width.

Як застосовується:

  • Коли більшість трафіку йде з десктопів.
  • Для складних веб-застосунків (CRM, ERP тощо), які зручніше використовувати на великих екранах.

Приклад коду (спочатку стилі для мобільних, потім для ширших екранів):

                        
                            body {
                                font-size: 20px;
                            }
                            
                            @media (max-width: 1024px) {
                                body {
                                    font-size: 18px;
                                }
                            }
                            
                            @media (max-width: 768px) {
                                body {
                                    font-size: 16px;
                                }
                            }

Adaptive Design (Адаптивний дизайн)

Суть: кожна версія сайту створюється окремо для різних пристроїв (мобільна, планшетна, десктопна).

Як застосовується:

  • Для сайтів, де потрібно сильно відрізняти дизайн на різних пристроях.
  • Наприклад, на мобільних може бути одна версія меню, а на десктопах — інша.
  • Використовуються серверні рішення для відправки різного HTML-коду залежно від пристрою.

Приклад: Google має окремі версії сайту для мобільних (m.google.com) і десктопних (google.com).

Responsive Design (Респонсивний дизайн)

Суть: сайт автоматично підлаштовується під будь-який екран завдяки гнучким сіткам, відсотковим значенням і медіа-запитам.

Як застосовується:

  • Найпоширеніший варіант для сучасних сайтів.
  • Легко підтримується, бо немає окремих версій для кожного пристрою.
  • Використовуються гнучкі сітки (Flexbox, Grid) та відсоткові розміри.

Приклад гнучкої верстки:

                        
                            .container {
                                width: 80%;
                                max-width: 1200px;
                                margin: 0 auto;
                            }
                            
                            .image {
                                width: 100%;
                                height: auto;
                            }

Що краще вибрати?

  • Якщо трафік мобільний → Mobile-First
  • Якщо проект складний, орієнтований на ПК → Desktop-First
  • Якщо потрібно окремі версії під кожен пристрій → Adaptive
  • Якщо хочеш універсальне рішення → Responsive

У сучасній розробці найчастіше використовують Mobile-First + Responsive Design!
